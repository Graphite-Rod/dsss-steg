# dsss-steg

Использование: dsss_cli <режим> <аргументы>
Режим embed:
	--container <имя файла>: Задаёт имя аудиофайла для использования как контейнер.
	--seed <число>: Число (ключ) для генерации последовательности PN.
	--chip <число>: Количество сэмплов, характеризующих один чип.
		Рекомендованные значения: 256, 512, 1024
	--lo_bin <число>: Нижний порог встраиваемых частот.
		Рекомендованные значения при chip=512: 150-260
	--hi_bin <число>: Верхний порог встраиваемых частот.
		Рекомендованные значения при chip=512: 300-450
	--secret <имя файла>: Имя файла, содержащего данные для встраивания.
	--out <имя файла>: Имя файла выходного стегоконтейнера.
Пример:
dsss_cli embed --container why-vbr2.mp3 --seed 42 --chip 512 --lo_bin 180 --hi_bin 400 --secret secret.txt --out why2.mp3

Режим extract:
	--container <имя файла>: Задаёт имя аудиофайла (контейнера) для извелечения.
	--seed <число>: Число (ключ) для генерации последовательности PN.
	--chip <число>: Количество сэмплов, характеризующих один чип.
	--lo_bin <число>: Нижний порог извлекаемых частот.
	--hi_bin <число>: Верхний порог извлекаемых частот.
	--out <имя файла>: Имя файла для сохранения извлечённых данных.
Пример:
dsss_cli extract --container why2.mp3 --seed 42 --chip 512 --lo_bin 180 --hi_bin 400 --out extracted.txt
